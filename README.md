# Neon Bar — ผนังรูปลูกค้าขึ้นจอทีวี 🎉

เว็บแอป (Next.js) ให้ลูกค้าสแกน QR แล้วส่งรูปขึ้นจอทีวีที่ร้านแบบเรียลไทม์
แปลงมาจากดีไซน์ `Neon Night - Variant A` ให้เป็นเว็บใช้งานได้จริง แยกเป็นหลายหน้า

## หน้า (routes)

| Path | ใช้ทำอะไร |
|------|-----------|
| `/` | หน้าแรก — ลิงก์ไปจอทีวี/หน้าส่งรูป + QR |
| `/tv` | **จอทีวี (กำแพงรูป)** — ขึ้นรูปสด ๆ แบบ realtime, โชว์ QR ให้สแกน |
| `/tv/slideshow` | **สไลด์โชว์เต็มจอ** — วนรูปทีละใบ |
| `/u` | **หน้ามือถือลูกค้า** — ถ่าย/เลือกรูป ใส่ชื่อ+ข้อความ แล้วส่ง |

## API

| Endpoint | หน้าที่ |
|----------|--------|
| `POST /api/photos` | อัปโหลดรูป (multipart: `image`, `name`, `msg`) — ผ่าน**ระบบคิว** |
| `GET /api/photos?since=<seq>` | ดึงเฉพาะรูปใหม่ (incremental) + `latestSeq` + `queue` depth |
| `DELETE /api/photos` | ล้างกำแพงรูป (เริ่มงานใหม่) |
| `GET /api/img/[id]` | เสิร์ฟไฟล์รูป |

รูปถูกเก็บไว้ในโฟลเดอร์ `data/` (gitignored) จึงอยู่ต่อแม้ restart

### Realtime ด้วย polling ทุก 3 วินาที

จอทีวี (`/tv`, `/tv/slideshow`) ยิง `GET /api/photos?since=<seq>` **ทุก 3 วินาที**
ส่งเฉพาะรูปที่ seq ใหม่กว่าที่เคยเห็น (เบา ไม่โหลดซ้ำ) และตรวจจับการล้างกำแพงได้
(`latestSeq` ตกลงต่ำกว่า cursor → resync)

### ระบบคิว (write queue) รองรับ burst

ทุกการอัปโหลด/ล้าง ต้องแก้ `photos.json` (read-modify-write) ถ้าหลายคนส่งพร้อมกัน
จะ race กันจนข้อมูลหาย — `WriteQueue` ใน [lib/store.ts](lib/store.ts) serialize งานเขียน
ให้ทำทีละรายการ จึงรองรับการส่งพร้อมกันจำนวนมากได้โดยไม่สูญหาย
(ทดสอบยิง 25 อัปโหลดพร้อมกัน → เก็บครบ 25, seq ไม่ซ้ำ, JSON ไม่พัง)
จอทีวีแสดง "กำลังประมวลผล N รูป…" และหน้ามือถือบอกจำนวนคิวเมื่อมีคนส่งพร้อมกัน

## วิธีรัน

```bash
npm install
npm run dev      # หรือ npm run build && npm start
```

### ⚠️ สำคัญ: ให้มือถือสแกน QR ได้

QR จะ encode URL ตาม host ที่เปิดหน้าจอทีวีอยู่ ดังนั้น
**เปิดหน้าจอทีวีผ่าน LAN IP ของเครื่อง ไม่ใช่ `localhost`** เช่น:

```
http://192.168.1.48:3000/tv
```

(Next.js จะ print `Network: http://192.168.x.x:3000` ตอนรัน)
มือถือต้องอยู่ Wi-Fi เดียวกัน ถึงจะสแกน QR แล้วเปิดหน้า `/u` ได้

ถ้ามีโดเมนจริง ตั้ง `NEXT_PUBLIC_BASE_URL=https://your-domain.com` เพื่อ override

## หมายเหตุทางเทคนิค

- รูปจากมือถือ (รวม HEIC ของ iPhone) ถูกแปลงเป็น JPEG + ย่อขนาดที่ฝั่ง browser
  ด้วย canvas ก่อนส่ง → จอทีวีแสดงได้ทุกเครื่อง และ payload เล็กลง
- Realtime ใช้ SSE + in-memory pub/sub — ทำงานบน server instance เดียว
  (เหมาะกับรันเครื่องเดียวที่ร้าน ไม่ใช่ serverless แบบ multi-node)
